GitHub Pages로 무료 블로그 호스팅하기: 설정부터 커스텀 도메인 연결까지

작성자: Starrope • 2026. 05. 30 • 읽는 시간: 약 5분
본문 상단 반응형 광고

워드프레스 호스팅 비용이 부담스럽고, 가비아나 카페24의 월정액을 내기 아깝다고 느낀 적 있으신가요? 개발자라면 이미 알고 있는 비밀, 바로 GitHub Pages를 활용하면 도메인 연결을 포함한 전문적인 개인 블로그를 완전히 무료로 운영할 수 있습니다. 전 세계 수백만 개발자와 블로거가 선택한 GitHub Pages는 단순히 무료라는 것을 넘어, 뛰어난 로딩 속도와 99.9% 업타임을 보장하는 엔터프라이즈급 CDN 인프라로 서비스됩니다. 이번 포스팅에서는 GitHub Pages 설정의 첫 단계부터 커스텀 도메인 연결까지 완전히 마스터합니다.

1. GitHub Pages란: 정적 웹사이트의 무료 호스팅 플랫폼

GitHub Pages는 GitHub 저장소(Repository)에 저장된 HTML, CSS, JavaScript 파일을 자동으로 웹사이트로 배포해 주는 서비스입니다. 서버 설정, 데이터베이스 관리, 보안 패치가 모두 GitHub에서 자동으로 처리되므로 콘텐츠 제작에만 집중할 수 있습니다.

  • 무료 HTTPS 제공: GitHub Pages는 Let's Encrypt 인증서를 자동으로 발급하여 무료 HTTPS를 적용합니다. 별도 SSL 인증서 구입이 불필요합니다.
  • 기본 도메인: GitHub Pages를 활성화하면 사용자명.github.io 형태의 기본 도메인이 자동 생성됩니다. 이 도메인만으로도 구글 검색 색인이 가능합니다.
  • 글로벌 CDN: GitHub의 CDN을 통해 전 세계 어디서나 빠른 로딩 속도를 제공합니다. 구글 Core Web Vitals 지표 개선에도 유리합니다.
"GitHub Pages는 단순한 무료 호스팅이 아닙니다. GitHub Actions와 결합하면 코드를 push하는 순간 자동으로 빌드·배포·CDN 갱신이 이루어지는 완전 자동화 파이프라인을 구축할 수 있습니다."

2. GitHub Pages 저장소 생성과 배포 활성화 단계별 가이드

GitHub Pages 설정은 생각보다 훨씬 간단합니다. GitHub 계정만 있으면 10분 안에 첫 번째 사이트를 온라인에 올릴 수 있습니다.

  • 저장소 생성: GitHub에서 새 저장소를 만들 때 이름을 반드시 사용자명.github.io 형식으로 설정하세요. 이 명명 규칙이 GitHub Pages 자동 활성화의 핵심입니다. 일반 저장소명으로 만들면 사용자명.github.io/저장소명 하위 경로로 배포됩니다.
  • 파일 업로드: index.html을 저장소의 최상위 루트에 push합니다. GitHub이 이 파일을 메인 진입점으로 자동 인식합니다. git init → git add . → git commit -m "first post" → git push 순서로 진행합니다.
  • Pages 설정 확인: 저장소 → Settings → Pages 메뉴에서 배포 소스 브랜치(보통 main)를 선택하고 Save를 클릭합니다. 1~3분 후 사용자명.github.io에 접속하면 사이트가 라이브 상태가 됩니다.

3. 커스텀 도메인 연결: 가비아·고대디 DNS 설정 완전 정복

기본 .github.io 도메인 대신 myblog.com처럼 나만의 브랜드 도메인을 연결하면 전문성이 크게 높아지고 SEO에도 유리합니다. 가비아나 고대디에서 도메인을 구입한 뒤 DNS 설정만 수정하면 됩니다.

  • A 레코드 설정: 도메인 등록처의 DNS 관리 패널에서 A 레코드를 GitHub Pages의 IP 주소 4개(185.199.108.153, 185.199.109.153, 185.199.110.153, 185.199.111.153)로 각각 추가합니다.
  • CNAME 파일 추가: 저장소 루트에 CNAME이라는 파일(확장자 없음)을 만들고 내용에 커스텀 도메인(myblog.com)만 입력 후 push합니다. GitHub이 이 파일을 인식해 도메인을 자동으로 연결합니다.
  • GitHub Settings에서 확인: Settings → Pages → Custom domain에 도메인을 입력하고 'Enforce HTTPS' 체크박스를 활성화합니다. DNS 전파에는 보통 수 분~24시간이 소요됩니다.

마치며

GitHub Pages는 디지털 노마드와 1인 크리에이터에게 최적의 블로그 플랫폼입니다. 서버 유지비 없이 전문적인 브랜드 블로그를 운영하고, 구글 애드센스를 통해 수익화까지 연결할 수 있습니다. 오늘 바로 GitHub 계정을 만들고 첫 번째 저장소를 생성해 보세요. HTML 파일 하나만 있으면 10분 안에 전 세계가 볼 수 있는 나만의 웹사이트가 완성됩니다.
