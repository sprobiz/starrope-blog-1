백엔드 개발 첫걸음: Node.js와 Express 프레임워크로 10분 만에 첫 웹 서버 구축하기

작성자: Starope • 2026. 06. 24 • 읽는 시간: 약 4분
본문 상단 반응형 광고

웹 개발을 시작하면 화면에 예쁜 버튼을 배치하고 텍스트 정렬을 담당하는 프론트엔드(Frontend) 영역을 거쳐, 마침내 데이터베이스와 소통하고 로그인 비즈니스 로직을 제어하는 백엔드(Backend, 서버) 개발 영역의 벽을 마주하게 됩니다. 과거에는 백엔드를 구현하기 위해 Java, PHP, C# 등 완전히 새로운 언어적 문법 체계와 복잡한 프레임워크 환경을 배워야 해서 비전공자 입문자들의 진입 장벽이 매우 높았습니다.

하지만 Node.js의 등장은 이 복잡한 생태계를 완벽하게 뒤흔들었습니다. 브라우저 안에서만 움직이던 자바스크립트(JavaScript) 엔진을 서버 컴퓨터 환경으로 격리 이식하여, 프론트엔드 개발자들이 쓰던 친근한 자바스크립트 문법 그대로 백엔드 서버 서버 코딩을 마스터할 수 있게 도왔기 때문입니다. 오늘은 이 Node.js 환경에 가장 널리 쓰이는 표준 경량 웹 프레임워크인 Express(익스프레스)를 결합해, 단 10분 만에 완벽히 구동하는 첫 백엔드 웹 서버를 구축하는 실전 로드맵을 알려드립니다.

1. Node.js 설치 및 프로젝트 기초 작업 패키지 초기화 (npm init)

서버 코딩을 시작하기 전 컴퓨터에 Node.js 런타임이 설치되어 있어야 합니다. 공식 홈페이지에서 안정성이 높은 LST 버전을 다운받아 설치한 뒤 터미널(CMD)을 열어 환경 구축 명령을 내립니다.

  • 프로젝트 폴더 생성 및 이동: 작업할 디렉토리를 생성하고 들어갑니다. (예: mkdir my-node-server && cd my-node-server)
  • NPM 패키지 초기화: npm init -y 명령을 입력합니다. 이 명령은 프로젝트의 뼈대가 되는 핵심 설정 파일인 package.json을 자동으로 생성하여 외부 라이브러리 연동 준비 상태를 구축합니다.
  • Express 라이브러리 설치: 터미널에 npm install express 명령을 실행해 경량 웹 서버 구축 프레임워크인 익스프레스를 안전하게 이식받습니다.
"Node.js는 자바스크립트의 영토를 서버로 확장시킨 혁명적인 런타임이며, 그 핵심 엔진 위에 얹어진 가장 가볍고 유연한 안장이 바로 Express입니다."

2. 단 10줄의 코드로 완성하는 익스프레스 서버 스크립트 작성

기초 환경 셋업이 끝났다면 프로젝트 루트 디렉토리에 app.js 라는 이름의 파일을 생성하고 아래의 핵심 자바스크립트 소스 코드를 기입합니다. 이것이 서버의 심장부가 될 소스입니다.

const express = require('express'); const app = express(); const PORT = 3000; app.get('/', (req, res) => { res.send('Hello World! 첫 노드 백엔드 서버가 완벽히 구동 중입니다.'); }); app.listen(PORT, () => { print(`Server is running on http://localhost:${PORT}`); });

작성을 마치고 터미널에 node app.js 명령을 내리는 순간, 여러분의 개인 컴퓨터는 3000번 포트를 활짝 열고 외부 클라이언트의 접속을 실시간으로 감시 대기하는 독립된 웹 서버 상태로 돌입합니다. 크롬 브라우저를 열고 주소창에 http://localhost:3000을 치면 우리가 보낸 응답 메시지가 출력되는 것을 마주할 수 있습니다.

3. REST API 구현: GET과 POST 요청을 통한 클라이언트 데이터 양방향 소통

단순 텍스트 출력을 넘어, 사용자가 입력한 데이터를 받고 다시 가공해서 돌려주는 REST API 엔드포인트를 설계하는 방법입니다.

  • 미들웨어(Middleware) 연동: 클라이언트가 보내는 JSON 형식의 정밀 데이터를 받아들이기 위해 서버 파일 최상단에 app.use(express.json()); 코드를 반드시 탑재시킵니다.
  • POST 요청 수신 처리: app.post('/api/users', (req, res) => { ... }) 구조를 빌드하여 유저 회원가입 데이터 등 민감한 페이로드를 가로채 데이터베이스 저장을 유도하는 백엔드 처리 로직을 구현합니다.

마치며

처음으로 서버가 구동되고 브라우저 주소창에 출력되는 "Hello World"를 마주했을 때의 짜릿한 쾌감은 모든 훌륭한 백엔드 개발자들이 거쳐온 공통된 성장의 기억입니다. 이제 여러분은 단순 화면 꾸미기(HTML/CSS)를 넘어, 데이터를 수집하고 보관하고 송출하는 **웹 서비스의 뇌신경계 시스템**에 입문하신 것입니다. 다음 포스팅에서는 실제 데이터베이스(MongoDB 혹은 MySQL)를 연동하는 법을 이어나가 보겠습니다. 서버 세팅 도중 에러가 나거나 막히는 명령이 있다면 언제든 편하게 질문 이메일을 보내주세요!
