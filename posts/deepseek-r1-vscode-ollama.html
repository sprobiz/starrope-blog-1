최근 생성형 AI 업계와 코딩 생태계를 가장 뜨겁게 달군 뉴스는 단연 중국의 스타트업이 개발한 오픈소스 추론형 AI 모델 DeepSeek-R1의 등장입니다. 이 모델은 성능적인 면에서 OpenAI의 최신 추론 모델인 o1과 대등하거나 더 나은 성능을 보여주면서도, 개발 비용은 단 10분의 1 수준으로 줄여 전 세계적인 주목을 받고 있습니다.

특히 DeepSeek-R1은 소스코드가 완전 오픈소스(MIT 라이선스)로 공개되어 사용자의 개인 로컬 PC에 직접 설치하여 무제한 무료로 구동할 수 있다는 강력한 장점이 있습니다. 이로 인해 사내 코드 유출 우려가 전혀 없는 안전한 코딩 환경을 구축할 수 있게 되었습니다. 이번 글에서는 로컬 머신 러닝 환경인 Ollama를 활용하여 DeepSeek-R1을 설치하고, 널리 사용되는 코드 편집기인 VS Code와 연동하여 진정한 무료 고성능 AI 코딩 비서를 구축하는 단계별 상세 가이드를 전해 드리겠습니다.

1. 로컬 AI 구동기 Ollama 설치 및 DeepSeek-R1 다운로드

로컬 PC에서 복잡한 설정 없이 인공지능 모델을 돌리기 위해 가장 널리 사용되는 도구가 바로 Ollama(올라마)입니다. Ollama는 CLI 환경에서 간단한 명령어로 수많은 오픈소스 거대 언어 모델(LLM)을 내려받고 서빙할 수 있도록 도와줍니다.

  • Ollama 웹사이트 접속: 먼저 Ollama 공식 웹사이트에 접속하여 윈도우(Windows), 맥(macOS) 또는 리눅스용 설치 파일을 다운로드하여 설치를 진행합니다.
  • PC 사양별 모델 크기 선택: DeepSeek-R1은 가볍게 튜닝된 1.5B(15억 파라미터) 모델부터 8B, 14B, 32B, 70B, 그리고 초대형 671B 풀 모델까지 다양하게 제공됩니다.
    • RAM 8GB ~ 16GB (내장 그래픽 등): deepseek-r1:1.5b 또는 deepseek-r1:8b 모델 권장.
    • RAM 16GB ~ 32GB (RTX 3060/4060급 GPU): deepseek-r1:14b 모델 권장 (속도와 추론 수준의 황금비율).
    • RAM 32GB 이상 (RTX 3090/4090급 전문 GPU): deepseek-r1:32b 또는 deepseek-r1:70b 고성능 추론 모델 적극 추천.
  • 모델 다운로드 명령 실행: 터미널(PowerShell 또는 CMD)을 열고 아래 명령어를 입력합니다: 
    ollama run deepseek-r1:8b
    명령을 입력하면 백그라운드에서 다운로드가 진행되며, 완료 후 터미널 창에서 즉시 대화가 가능한 프롬프트가 실행됩니다.
"내 로컬 PC 하드웨어 리소스를 사용하므로, 한 달에 수만 원씩 지불해야 하던 ChatGPT나 Claude 구독료를 완벽히 0원으로 아낄 수 있습니다."

2. VS Code 최강의 오픈소스 AI 확장 프로그램 'Continue' 설치

Ollama를 통해 DeepSeek-R1 모델이 정상적으로 실행된다면, 이제 이를 코드 편집기인 VS Code와 다리(Bridge)로 연결해 주어야 합니다. 이때 가장 적합한 도구가 바로 Continue(컨티뉴) 확장 프로그램입니다. GitHub Copilot과 유사한 코딩 인터페이스를 100% 무료 오픈소스로 제공합니다.

  1. VS Code를 열고 좌측 사이드바의 확장(Extensions) 아이콘을 클릭합니다.
  2. 검색창에 'Continue'를 검색한 뒤, Continue - Llama 3, DeepSeek, and more 항목을 설치합니다.
  3. 설치가 완료되면 VS Code 왼쪽 사이드 바에 작은 원형 아이콘(Continue 전용 탭)이 추가됩니다.

3. VS Code Continue의 config.json 설정 및 연동 완료하기

설치된 Continue 확장이 로컬에 떠 있는 Ollama의 DeepSeek-R1 API 엔드포인트를 바라보도록 설정 파일을 간단히 편집해 주어야 합니다.

Continue 패널 하단의 톱니바퀴 아이콘(Configure)을 누르면 설정 파일인 config.json이 팝업됩니다. 이 파일의 models 배열 영역을 다음과 같이 수정합니다:

{
  "models": [
    {
      "title": "DeepSeek R1 8B",
      "provider": "ollama",
      "model": "deepseek-r1:8b"
    }
  ],
  "tabAutocompleteModel": {
    "title": "DeepSeek Coder 1.5B",
    "provider": "ollama",
    "model": "deepseek-coder:1.5b"
  }
}

위와 같이 연동한 뒤 설정을 저장하면 연동 작업이 완료됩니다. 이제 VS Code 내에서 Ctrl + L(Mac은 Cmd + L)을 눌러 사이드바 챗 창에 "React에서 무한 스크롤 구현하는 컴포넌트 코드 짜줘"라고 대화하면, 로컬의 DeepSeek-R1이 소스코드를 막힘없이 추론하며 생성하는 신세계를 경험하실 수 있습니다. 특히 코드 편집 중에 필요한 영역을 선택한 후 Ctrl + I를 눌러 인라인 코드 편집 및 자동완성을 지시하는 것도 완벽히 지원됩니다.

오늘 소개해 드린 Ollama + DeepSeek-R1 + VS Code Continue 조합은 비싼 구독 비용과 보안 유출 걱정이라는 두 마리 토끼를 단번에 잡은 차세대 개발 환경의 바이블입니다. 물론 로컬 AI의 처리 속도는 개인 PC 하드웨어 성능(특히 GPU 그래픽카드 탑재 메모리 크기 VRAM)에 크게 좌우되므로, 본인의 시스템에 최적화된 파라미터 크기의 모델을 이것저것 테스트하며 적용하는 것을 권장합니다. 코딩 도중 연동 에러가 발생하거나 사양 튜닝 설정에 추가 질문이 있다면 편하게 이메일이나 피드백 댓글을 남겨주세요!

❓ 자주 묻는 질문 (FAQ)

DeepSeek-R1 로컬 설치에 그래픽카드(GPU)가 필수인가요?

필수는 아닙니다. CPU와 RAM 리소스만으로도 구동은 가능하지만, 추론 속도가 현저히 느려집니다. 쾌적한 코딩 답변을 얻기 위해서는 엔비디아(NVIDIA) RTX 계열이나 맥북 실리콘 칩(M1, M2, M3 등)이 탑재된 시스템에서 GPU 가속을 사용하는 것이 가장 좋습니다.

DeepSeek-R1 로컬 환경 구축 시 회사 기밀 코드 유출 우려가 없나요?

네, 완벽히 안전합니다. Ollama를 활용한 로컬 환경 구축은 네트워크 연결 없이 오프라인 상태에서도 100% 동작합니다. 즉, 작성한 모든 소스 코드와 프롬프트 대화는 외부 서버(OpenAI, Anthropic 등)로 절대 전송되지 않고 내 PC 하드 디스크와 램 안에서만 처리됩니다.

자동완성(Tab autocomplete) 속도가 너무 느린데 해결 방법이 있나요?

자동완성 모델은 코드를 작성하는 매 순간 백그라운드에서 실행되므로 훨씬 작고 가벼운 모델을 설정해야 합니다. `config.json`의 `tabAutocompleteModel` 영역에 `deepseek-coder:1.5b` 또는 `qwen2.5-coder:1.5b`처럼 15억 파라미터 이하급 초경량 코딩 특화 모델을 지정해 두시면 속도가 극적으로 빨라집니다.
