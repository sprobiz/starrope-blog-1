마크다운(Markdown) 완벽 마스터: 개발자와 블로거를 위한 핵심 문법 총정리

작성자: Starrope • 2026. 06. 26 • 읽는 시간: 약 5분
본문 상단 반응형 광고

깃허브(GitHub)에서 프로젝트 README를 작성하거나, 노션(Notion)에서 팀 문서를 정리하거나, 기술 블로그에 글을 쓸 때 공통적으로 사용되는 언어가 있습니다. 바로 마크다운(Markdown)입니다. 2004년 존 그루버(John Gruber)가 만든 이 경량 마크업 언어는 HTML의 복잡한 태그 없이도 아름다운 문서를 만들 수 있어, 전 세계 개발자와 콘텐츠 크리에이터의 표준 도구가 되었습니다. 이번 포스팅에서는 실무에서 자주 쓰이는 마크다운 핵심 문법을 빠짐없이 정리합니다.

1. 텍스트 강조와 헤딩(Heading) 계층 구조 설계

문서의 논리적 계층을 정의하는 헤딩은 SEO와 문서 가독성 모두에 직결됩니다. 샵(#) 기호의 개수로 제목 레벨(H1~H6)을 결정합니다. H1은 문서 전체의 핵심 주제를 표현하며 페이지당 하나만 사용하는 것이 SEO 황금 원칙입니다.

  • 볼드체 강조: 별표 두 개로 감싸면 **굵은 텍스트** → 굵은 텍스트가 됩니다. 핵심 키워드 강조에 사용합니다.
  • 이탤릭체 강조: 별표 하나로 감싸면 *기울어진 텍스트* → 용어 설명이나 외래어 표기에 적합합니다.
  • 취소선: 물결표 두 개로 감싸면 ~~삭제된 텍스트~~ → 변경 사항을 시각적으로 표시할 때 활용합니다.
  • 인라인 코드: 백틱(`)으로 감싸면 명령어나 변수명을 코드 형태로 표시할 수 있습니다.
"마크다운은 '읽기 쉬운 평문'을 최우선 가치로 설계되었습니다. 마크다운 파일은 렌더링 없이 일반 텍스트로 봐도 내용을 파악할 수 있어야 합니다."

2. 리스트와 체크박스: 업무 관리부터 기술 문서까지

마크다운의 리스트는 단순한 나열을 넘어 프로젝트 TODO 관리와 문서 구조화에 핵심적으로 활용됩니다. 깃허브 이슈와 PR 설명에서 체크박스 리스트는 진행 상황을 한눈에 보여주는 강력한 도구입니다.

  • 순서 없는 리스트: 하이픈(-), 별표(*), 플러스(+) 중 하나로 시작합니다. 두 칸 들여쓰기로 중첩 리스트를 만들 수 있습니다.
  • 순서 있는 리스트: 숫자와 마침표(1. 2. 3.)로 시작합니다. 실제 번호가 틀려도 렌더링 시 자동으로 순서가 정렬됩니다.
  • GitHub 체크박스: - [ ] 미완료, - [x] 완료 형태로 작성하면 클릭 가능한 체크박스가 생성됩니다. 스프린트 태스크 관리에 필수입니다.

3. 코드 블록과 문법 강조(Syntax Highlighting)

기술 블로그와 개발 문서에서 가장 중요한 마크다운 요소는 단연 코드 블록입니다. 백틱 세 개(```)로 시작하고 끝내는 펜스드 코드블록(Fenced Code Block)은 언어 이름을 함께 명시하면 색상 강조(Syntax Highlighting)가 자동 적용됩니다.

  • JavaScript 예시: 시작 부분에 ```javascript를 명시하면 JS 문법에 맞게 키워드, 문자열, 주석이 색깔로 구분됩니다.
  • Python, Bash, SQL 등: python, bash, sql 등 수십 가지 언어를 지원합니다. 명령줄 예시는 bash로 표기하는 것이 관례입니다.
  • diff 형식: ```diff를 사용하면 변경 전(-)과 변경 후(+)를 빨강/초록으로 시각화하여 코드 리뷰에 최적화된 뷰를 제공합니다.

4. 표(Table)와 링크: 정보를 구조화하는 최강 조합

마크다운 표는 파이프(|) 기호와 하이픈(-)으로 직관적으로 구성됩니다. 기술 비교, API 파라미터 설명, 단가 비교 등에서 정보를 한눈에 전달하는 데 탁월합니다. 콜론(:) 위치로 좌·우·중앙 정렬을 자유롭게 지정할 수 있습니다.

  • 인라인 링크: [링크 텍스트](URL) 형식으로 외부 페이지나 내부 앵커(#section)로 연결합니다. 텍스트 안에 자연스럽게 녹아드는 링크를 만들 때 사용합니다.
  • 참조형 링크: 본문에는 [텍스트][ref]를, 문서 하단에 [ref]: URL을 선언합니다. 긴 URL이 본문 가독성을 해칠 때 유용합니다.
  • 이미지 삽입: 링크 앞에 느낌표(!)를 붙이면 ![대체 텍스트](이미지URL)로 이미지를 삽입합니다. alt 텍스트는 SEO와 접근성 모두에 필수입니다.

마치며

마크다운은 한 번 손에 익으면 다시는 리치 텍스트 에디터로 돌아가기 싫어질 만큼 편리합니다. 깃허브 README에서 시작해 개인 블로그, 팀 위키, 기술 문서화까지 마크다운이 쓰이지 않는 곳이 없습니다. 오늘 배운 헤딩, 리스트, 코드블록, 표, 링크 다섯 가지 핵심 문법만 완벽히 익혀도 전문적인 개발자 문서를 자유롭게 작성할 수 있습니다. 마크다운 에디터로는 Typora, Obsidian, VS Code를 추천하며, 깃허브에서 바로 미리보기를 확인하며 실습하는 것이 가장 빠른 습득 경로입니다.
