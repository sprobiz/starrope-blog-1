React 입문 완전 가이드: 컴포넌트·State·Props로 동적 웹앱 만들기

작성자: Starrope • 2026. 06. 27 • 읽는 시간: 약 6분
본문 상단 반응형 광고

스택오버플로우(Stack Overflow) 개발자 설문 조사에서 수년째 '가장 많이 사용되는 웹 프레임워크' 1위를 차지하고 있는 React(리액트)는 메타(구 페이스북)가 2013년 오픈소스로 공개한 UI 라이브러리입니다. 단순히 인기가 많은 것을 넘어, 넷플릭스, 에어비앤비, 인스타그램 등 전 세계 최대 규모의 서비스들이 React로 구축되어 있습니다. 이번 포스팅에서는 React가 왜 특별한지부터, 핵심 개념인 컴포넌트·State·Props·훅(Hook)까지 완전히 정복합니다.

1. 왜 React인가: 컴포넌트 기반 UI 개발의 혁명

기존의 전통적인 웹 개발 방식에서는 HTML, CSS, JavaScript가 분리된 파일에 따로 존재했습니다. React는 이 패러다임을 완전히 뒤집어 컴포넌트(Component)라는 개념을 도입했습니다. 하나의 컴포넌트는 특정 UI 영역의 HTML 구조(JSX), 스타일, 동작 로직을 하나의 함수 안에 캡슐화합니다.

  • 재사용성(Reusability): 한 번 만든 버튼, 카드, 모달 컴포넌트를 앱 전체 어디서든 불러와 사용합니다. 코드 중복이 사라져 유지보수가 극적으로 쉬워집니다.
  • 가상 DOM(Virtual DOM): React는 실제 DOM을 직접 조작하지 않고, 메모리 속 가상의 DOM 트리를 먼저 수정한 후 변경된 부분만 실제 DOM에 적용합니다. 이 방식이 React의 놀라운 렌더링 성능 비결입니다.
  • 단방향 데이터 흐름: 데이터는 항상 부모 컴포넌트에서 자식 컴포넌트로 Props를 통해 흐릅니다. 이 예측 가능한 데이터 흐름이 복잡한 앱에서도 디버깅을 쉽게 만듭니다.
"React를 배우는 것은 단순히 라이브러리 하나를 익히는 게 아닙니다. 컴포넌트 기반으로 생각하는 방식, 즉 UI를 설계하는 새로운 철학을 습득하는 과정입니다."

2. JSX: JavaScript 안에서 HTML을 작성하는 마법의 문법

JSX(JavaScript XML)는 React의 트레이드마크라 할 수 있는 독특한 문법입니다. 얼핏 보면 HTML처럼 보이지만, 실제로는 JavaScript 코드입니다. Babel 같은 트랜스파일러가 JSX를 브라우저가 이해할 수 있는 순수 JavaScript 함수 호출로 변환해 줍니다.

  • 중괄호로 JS 표현식 삽입: JSX 안에서 {변수명} 또는 {함수()} 형태로 JavaScript 표현식을 HTML 속에 직접 주입할 수 있습니다. 동적 데이터를 UI에 렌더링하는 핵심 방법입니다.
  • className vs class: HTML의 class 속성은 JSX에서 className으로 써야 합니다. class는 JS 예약어이기 때문입니다. for는 htmlFor로 사용합니다.
  • 단일 루트 요소: 컴포넌트가 반환하는 JSX는 반드시 하나의 최상위 부모 태그로 감싸야 합니다. 불필요한 <div> 추가를 피하고 싶다면 빈 태그 <></>(Fragment)를 활용하세요.

3. useState와 useEffect: 함수형 컴포넌트를 살아 숨 쉬게 하는 핵심 훅

2019년 React 16.8 버전에서 도입된 훅(Hook)은 함수형 컴포넌트에서 상태(State)와 생명주기(Lifecycle)를 관리할 수 있게 해주는 혁신적인 API입니다. 복잡한 클래스형 컴포넌트를 대체하여 React 개발을 훨씬 간결하게 만들었습니다.

  • useState: 컴포넌트 내부에서 변하는 데이터를 관리합니다. const [count, setCount] = useState(0)처럼 선언하면, count는 현재 값, setCount는 값을 갱신하는 함수입니다. 값이 바뀔 때마다 컴포넌트가 자동으로 리렌더링됩니다.
  • useEffect: 컴포넌트가 렌더링된 후 실행되는 사이드 이펙트(API 호출, 타이머, 이벤트 리스너 등록 등)를 처리합니다. 두 번째 인자로 전달하는 의존성 배열(dependency array)로 실행 타이밍을 정밀하게 제어합니다.
  • Props 전달: 부모 컴포넌트에서 <Button label="클릭" onClick={handleClick} />처럼 속성을 전달하면, 자식 컴포넌트는 props.label, props.onClick으로 받아 사용합니다.

4. 첫 React 프로젝트 세팅: Vite로 5분 만에 시작하기

과거에는 create-react-app이 공식 시작 도구였지만, 현재는 Vite(비트)가 개발 서버 속도와 빌드 성능에서 압도적 우위를 보여 사실상의 표준으로 자리잡았습니다. 터미널에 npm create vite@latest my-app -- --template react를 입력하면 즉시 React + Vite 프로젝트가 생성됩니다.

  • 파일 구조 이해: src/App.jsx가 메인 컴포넌트, src/main.jsx가 React를 DOM에 마운트하는 진입점입니다. src/components/ 폴더를 만들어 재사용 컴포넌트를 관리하는 것이 관례입니다.
  • 개발 서버 실행: npm run dev 명령으로 핫 모듈 교체(HMR)가 지원되는 빠른 로컬 개발 서버가 실행됩니다. 코드를 수정하면 브라우저가 즉시 반응합니다.
  • React DevTools: 크롬 확장 프로그램 'React Developer Tools'를 설치하면 컴포넌트 트리, State, Props 값을 실시간으로 검사할 수 있습니다. 디버깅의 필수 무기입니다.

마치며

React는 진입 장벽이 있지만, 컴포넌트·State·Props·훅이라는 네 가지 핵심 개념을 이해하면 그 이후로는 놀랍도록 논리적이고 즐거운 개발 경험이 펼쳐집니다. React를 통해 웹 개발자로서 취업 시장과 프리랜서 시장 모두에서 경쟁력을 급격히 높일 수 있습니다. 공식 문서인 react.dev는 한국어로도 제공되어 체계적인 학습에 최적의 출발점이 됩니다. 오늘 당장 Vite로 첫 프로젝트를 만들고, 간단한 할 일 목록(To-Do List) 앱을 스스로 구현해 보세요.
